# GitHub Copilot Instructions for This Project

## Angular Naming Conventions

When suggesting or generating Angular files, ALWAYS follow these naming conventions:

### Components
- File: `example.component.ts` (NOT `example.ts`)
- Template: `example.component.html`
- Styles: `example.component.scss`
- Spec: `example.component.spec.ts`
- Class: `ExampleComponent` (MUST end with `Component`)

### Services
- File: `example.service.ts` (NOT `example.ts`)
- Class: `ExampleService` (MUST end with `Service`)

### Modules
- File: `example.module.ts`
- Routing: `example-routing.module.ts`
- Class: `ExampleModule`

### Other Files
- Directives: `*.directive.ts`
- Pipes: `*.pipe.ts`
- Guards: `*.guard.ts`

## File Naming Style
- Use kebab-case for file names
- Always include the type suffix (component, service, module, etc.)

## Class Naming Style
- Use PascalCase
- Always include the type suffix (Component, Service, Module, etc.)

## Important
If you see files without proper suffixes, suggest adding them immediately.
